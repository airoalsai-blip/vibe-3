# Claude Code 첫 작업 프롬프트

아래 내용을 Claude Code에 그대로 넣어서 첫 작업으로 시작한다.

## 작업 목표

날씨 특보 기반 비상근무 알림 시스템을 구현한다.

현재 우선순위는 기능 구현보다 구조 확정이다.

먼저 다음 문서를 읽고, 설계상 빠진 부분이 있으면 제안한 뒤 구현 순서를 잡는다.

- `weather/README.md`
- `weather/01_screen_design.md`
- `weather/02_database_schema.md`

## 기술 조건

- FE: HTML + JavaScript + CSS
- BE: Python + FastAPI
- DB: SQLite
- 모든 UI 문구는 한글로 표시
- 처음 쓰는 사람도 이해하기 쉬운 관리 화면으로 구성

## 핵심 요구사항

1. 날씨 API를 5분마다 조회해서 특보 정보를 갱신한다.
2. 지역을 선택할 수 있어야 한다.
3. 비상근무 명단은 파일 업로드 또는 수기 입력으로 등록할 수 있어야 한다.
4. 파일 형식은 `xlsx`, `csv`, `hwp`, `hwpx`, `pdf`를 고려한다.
5. 부서별로 어떤 특보를 알림 대상으로 받을지 체크박스로 저장한다.
6. 저장한 정책은 나중에 수정할 수 있어야 한다.
7. 순번이 지나면 다음 순번에게 문자 또는 카카오톡으로 사전 알림을 보낸다.
8. 선택한 날짜에 특보가 발생하면 해당 순번에게 즉시 알림을 보낸다.
9. 모든 발송과 수집 이력은 DB에 저장한다.

## Claude Code 작업 순서

1. 문서 기준으로 MVP 범위를 재정리한다.
2. UI 화면 구성을 제안한다.
3. SQLite 테이블 초안을 검토하고 필요하면 수정한다.
4. FastAPI 프로젝트 구조를 제안한다.
5. 먼저 README, 화면 설계, DB 설계를 완성한다.
6. 그 다음 FE와 BE 스캐폴딩을 만든다.
7. 마지막으로 날씨 API, 명단 업로드, 정책 저장, 알림 로직 순서로 구현한다.

## 구현 원칙

- 한 번에 크게 만들지 말고 단계적으로 만든다.
- 사용자가 이해하기 쉬운 화면을 우선한다.
- 설정 값은 나중에 수정 가능해야 한다.
- 중복 알림을 막는 구조를 먼저 넣는다.
- 실제 외부 문자/카카오톡 연동은 추상화 레이어로 분리한다.
