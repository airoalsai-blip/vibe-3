# 비상근무 알림 시스템 화면 설계서

## 1. 설계 목표

- 처음 보는 사용자도 바로 쓸 수 있게 단순한 구조로 만든다.
- 관리자는 지역, 명단, 정책, 알림을 한 화면 흐름으로 관리할 수 있어야 한다.
- 특보 현황과 알림 대상이 한눈에 보이도록 한다.
- 수정이 자주 필요한 항목은 바로 편집할 수 있게 한다.

## 2. 전체 화면 구조

1. 대시보드
2. 지역 선택 및 특보 현황
3. 직원 명단 관리
4. 부서별 특보 정책 관리
5. 순번 및 알림 관리
6. 발송 이력

## 3. 주요 화면 상세

### 3.1 대시보드

목적:
- 현재 시스템 상태를 빠르게 파악한다.

주요 요소:
- 선택 지역
- 최신 특보 카드
- 오늘의 비상근무 대상자
- 다음 순번 알림 예정자
- 최근 발송 내역
- 수동 새로고침 버튼

사용자 행동:
- 지역 변경
- 특보 상세 확인
- 알림 대상 확인
- 명단/정책 화면으로 이동

### 3.2 지역 선택 화면

목적:
- 특보를 확인할 지역을 선택한다.

주요 요소:
- 시/도 선택
- 시/군/구 선택
- 즐겨찾기 지역
- 선택 지역 저장 버튼

동작:
- 지역을 선택하면 즉시 특보 조회 기준이 바뀐다.
- 지역은 기본값으로 저장된다.

### 3.3 특보 현황 화면

목적:
- 선택 지역의 특보를 한눈에 보여준다.

주요 요소:
- 특보 종류
- 발효 시각
- 종료 시각
- 영향 지역
- 경고 수준
- 수집 시각

표현 방식:
- 강풍주의보: 노란색 계열
- 호우주의보: 주황색 계열
- 호우경보: 빨간색 계열
- 특보 없음: 회색 계열

### 3.4 직원 명단 관리 화면

목적:
- 비상근무 대상자를 등록하고 수정한다.

주요 요소:
- 파일 업로드 버튼
- 수기 입력 버튼
- 미리보기 테이블
- 명단 검색
- 부서 필터
- 순번 수정
- 행 단위 편집/삭제

파일 업로드 흐름:
1. 파일 선택
2. 형식 검사
3. 내용 미리보기
4. 컬럼 매핑
5. 저장

### 3.5 부서 정책 관리 화면

목적:
- 부서별로 어떤 특보를 알림 대상으로 볼지 설정한다.

주요 요소:
- 부서 선택
- 지역 연결
- 특보 유형 체크박스
- 알림 활성/비활성
- 저장 버튼
- 수정 버튼

특보 체크박스 예시:
- 강풍주의보
- 강풍경보
- 호우주의보
- 호우경보
- 대설주의보
- 대설경보

동작:
- 체크한 항목만 알림 대상이 된다.
- 저장 후에도 언제든 수정 가능해야 한다.

### 3.6 순번 및 알림 관리 화면

목적:
- 비상근무 순번과 사전 알림, 즉시 알림을 관리한다.

주요 요소:
- 현재 순번
- 다음 순번
- 사전 알림 시점
- 알림 채널 선택
- 즉시 알림 버튼
- 예약 알림 목록

알림 채널:
- 문자
- 카카오톡
- 둘 다

### 3.7 발송 이력 화면

목적:
- 어떤 알림이 누구에게 언제 갔는지 확인한다.

주요 요소:
- 발송 시각
- 대상자
- 부서
- 특보 유형
- 지역
- 채널
- 성공 여부
- 재시도 여부

## 4. 공통 UI 원칙

- 모든 라벨과 버튼은 한글로 표시한다.
- 저장, 수정, 삭제 버튼은 색을 다르게 둔다.
- 위험 정보는 빨간색 계열로 강조한다.
- 중요한 설정은 토글보다 체크박스와 명확한 저장 버튼을 사용한다.
- 관리자용 화면은 단계별로 나누지 말고 한 화면에서 흐름이 보이게 한다.

## 5. 추천 레이아웃

### 상단

- 로고/서비스명
- 선택 지역
- 마지막 수집 시각
- 로그인 사용자

### 좌측 메뉴

- 대시보드
- 지역 관리
- 명단 관리
- 정책 관리
- 알림 관리
- 발송 이력

### 본문

- 요약 카드
- 상세 테이블
- 설정 폼
- 로그 목록

## 6. UX 포인트

- 업로드 후 바로 미리보기
- 특보 발생 시 색상 경고
- 정책 수정은 원래 설정을 다시 불러와 편집
- 명단 추가는 행 추가 방식도 제공
- 순번 변경은 드래그보다 입력 방식 우선
