1. 개요

공공직군 행정업무 슈퍼앱은 팀 단위 행정업무를 한곳에서 처리하기 위한 내부 업무 플랫폼이다. 초기 범위는 팀원 스케줄 관리, 엑셀 업무 자동화, 민원 대응 챗봇, 뉴스 기사 수집이다.

핵심 기능

  • 팀원 휴가, 근무, 출장 일정을 공유하는 캘린더
  • 특정 컬럼 기준 엑셀 분할 및 여러 파일 병합
  • 첨부 민원 매뉴얼 기반 답변 초안 및 스크립트 생성
  • 공공 행정 관련 뉴스의 매일 아침 자동 수집

주요 사용자

  • 팀원: 일정 등록, 엑셀 처리, 민원 답변 초안 생성
  • 팀장: 팀 일정과 업무 부하 확인
  • 민원 담당자: 매뉴얼 기반 대응 품질 표준화
  • 시스템 관리자: 사용자, 권한, 매뉴얼, 뉴스 키워드 관리

2. PRD 요약

제품 목표

  • 반복적인 행정업무 처리 시간을 줄인다.
  • 팀원 일정과 부재 정보를 투명하게 공유한다.
  • 민원 대응 문구를 매뉴얼 기반으로 표준화한다.
  • 매일 확인해야 하는 공공 행정 뉴스를 자동 브리핑한다.

MVP 범위

기능 MVP 범위 수용 기준
스케줄 관리 일정 등록, 조회, 수정, 삭제, 월간/주간 보기 유형별 표시와 팀 단위 조회가 가능해야 한다.
엑셀 자동화 업로드, 미리보기, 기준 컬럼 분할, 파일 병합 분할 결과 다운로드와 컬럼 불일치 경고를 제공해야 한다.
민원 챗봇 매뉴얼 업로드, 색인, 질문 답변, 근거 표시 근거 없는 답변은 불확실성을 표시해야 한다.
뉴스 수집 키워드 기반 수집, 중복 제거, 요약, 목록 표시 원문 링크와 발행 정보를 제공해야 한다.

성공 지표

지표 목표
엑셀 처리 시간수작업 대비 70% 이상 단축
일정 누락 문의도입 전 대비 50% 이상 감소
민원 답변 초안 작성 시간평균 50% 이상 단축
뉴스 브리핑 확인율업무일 기준 80% 이상

3. Architecture 요약

권장 기술 스택

영역 기술 역할
FrontendReact, TypeScript, Vite업무 화면과 사용자 인터랙션
BackendPython FastAPIAPI, 권한, 업무 로직
DatabasePostgreSQL사용자, 일정, 이력, 뉴스 메타데이터
QueueRedis, Celery 또는 RQ엑셀 처리, 문서 색인, 뉴스 수집
AI/RAG임베딩 모델, 벡터 DB, LLM API매뉴얼 기반 민원 답변 생성

시스템 구성

사용자 브라우저
  -> React/Vite Frontend
  -> FastAPI Backend
     -> PostgreSQL
     -> File Storage
     -> Worker Queue
     -> Vector Store
     -> LLM API / 뉴스 API / RSS

모듈 구분

  • calendar: 팀원 일정과 부재 현황 관리
  • excel: 파일 업로드, 미리보기, 분할, 병합
  • chatbot: 매뉴얼 검색, 답변 초안, 근거 표시
  • news: 키워드 관리, 기사 수집, 요약, 브리핑
  • admin: 사용자, 권한, 설정, 감사 로그 관리

4. Operation 요약

로컬 실행 예시

Frontend

cd frontend
npm.cmd install
npm.cmd run dev

Backend

cd backend
.\.venv\Scripts\python.exe -m app.main

운영 체크리스트

  • 환경 변수와 비밀키 설정
  • 데이터베이스 마이그레이션
  • 관리자 계정 생성
  • 민원 매뉴얼 초기 업로드 및 색인
  • 뉴스 수집 스케줄 확인
  • 파일 업로드 용량 제한과 감사 로그 확인
  • 백업 작업과 장애 알림 채널 설정

자주 발생하는 오류

증상 대응
npm run dev 없음package.json에 Vite 개발 스크립트를 추가한다.
엑셀 분할 실패기준 컬럼명, 빈 값, 병합 셀 여부를 확인한다.
챗봇 근거 없음매뉴얼 색인 상태와 최신 문서 업로드 여부를 확인한다.
뉴스 미수집스케줄러, 외부 API, 키워드 설정을 확인한다.
