# 공공직군 행정업무 슈퍼앱 PRD

## 1. 문서 목적

본 문서는 공공직군 행정업무 슈퍼앱의 제품 요구사항을 정의한다. 구현 전 이해관계자가 기능 범위, 사용자 흐름, 우선순위, 성공 기준을 동일하게 이해하는 것을 목표로 한다.

## 2. 제품 개요

공공직군 행정업무 슈퍼앱은 팀 단위 행정업무를 한곳에서 처리하도록 돕는 내부 업무 플랫폼이다. 주요 기능은 팀원 스케줄 공유, 엑셀 업무 자동화, 민원 대응 챗봇, 공공 행정 뉴스 수집이다.

### 핵심 가치

- 반복적인 행정업무 처리 시간을 줄인다.
- 팀원 일정과 업무 현황을 투명하게 공유한다.
- 민원 대응 품질을 표준화한다.
- 매일 확인해야 하는 공공 행정 뉴스를 자동으로 정리한다.

## 3. 대상 사용자

| 사용자 | 설명 | 주요 니즈 |
| --- | --- | --- |
| 팀원 | 일반 행정 담당자 | 일정 등록, 엑셀 처리, 민원 답변 초안 생성 |
| 팀장 | 팀 운영 책임자 | 팀 일정 확인, 업무 부하 파악, 이슈 대응 |
| 민원 담당자 | 민원 응대 실무자 | 매뉴얼 기반 대응 문구 작성, 유사 사례 확인 |
| 시스템 관리자 | 서비스 운영 담당자 | 사용자 권한, 뉴스 수집, 매뉴얼 관리, 장애 대응 |

## 4. 기능 범위

### 4.1 팀원 스케줄 관리

팀원들의 휴가, 근무, 출장, 교육, 재택, 외근 등 일정을 공유하는 캘린더 기능이다.

#### 주요 기능

- 월간, 주간, 일간 캘린더 보기
- 팀원별 일정 필터링
- 일정 유형별 색상 표시
- 휴가, 출장, 근무, 교육, 기타 일정 등록
- 일정 반복 설정
- 일정 변경 및 삭제
- 팀 전체 일정 검색
- 권한에 따른 일정 열람 범위 제어

#### 사용자 시나리오

1. 팀원이 본인 휴가 일정을 등록한다.
2. 팀장은 주간 캘린더에서 팀원 부재 현황을 확인한다.
3. 민원 담당자는 출장 중인 담당자를 확인하고 대체 담당자에게 업무를 배정한다.

#### 수용 기준

- 사용자는 일정 유형, 시작일시, 종료일시, 제목, 설명을 입력해 일정을 등록할 수 있어야 한다.
- 캘린더는 최소 월간/주간 보기 전환을 지원해야 한다.
- 일정 유형은 색상 또는 아이콘으로 구분되어야 한다.
- 본인이 등록한 일정은 수정/삭제할 수 있어야 한다.
- 관리자 또는 팀장은 팀 전체 일정을 조회할 수 있어야 한다.

### 4.2 엑셀 업무 자동화

업로드된 엑셀 파일을 특정 컬럼 기준으로 나누거나 여러 파일을 하나로 합치는 기능이다.

#### 주요 기능

- `.xlsx`, `.xls`, `.csv` 파일 업로드
- 파일 미리보기
- 기준 컬럼 선택
- 기준 컬럼 값별 파일 분할
- 여러 엑셀 파일 병합
- 헤더 행 지정
- 빈 행 제거
- 중복 행 제거 옵션
- 처리 결과 다운로드
- 처리 이력 저장

#### 사용자 시나리오

1. 담당자가 민원 접수 목록 엑셀을 업로드한다.
2. `부서명` 컬럼을 선택해 부서별 파일로 분할한다.
3. 시스템은 분할된 파일을 압축 파일로 제공한다.
4. 담당자는 여러 부서 취합 파일을 업로드해 하나의 통합 파일로 병합한다.

#### 수용 기준

- 사용자는 업로드한 파일의 컬럼명을 확인할 수 있어야 한다.
- 사용자는 분할 기준 컬럼을 선택할 수 있어야 한다.
- 분할 결과 파일명은 기준 컬럼 값을 포함해야 한다.
- 병합 시 컬럼 불일치 파일은 경고로 표시되어야 한다.
- 대용량 파일 처리 중 진행 상태를 확인할 수 있어야 한다.

### 4.3 민원 대응 챗봇

첨부된 민원 매뉴얼을 기반으로 민원 대응 방식, 답변 초안, 전화 응대 스크립트를 작성해주는 챗봇이다.

#### 주요 기능

- 민원 매뉴얼 파일 업로드
- 매뉴얼 문서 색인
- 사용자 질문 입력
- 관련 매뉴얼 조항 검색
- 답변 초안 생성
- 전화 응대 스크립트 생성
- 공문/이메일 형식 문안 생성
- 답변 근거 문서 표시
- 부적절하거나 불확실한 답변 경고
- 민감정보 입력 주의 안내

#### 사용자 시나리오

1. 관리자가 민원 대응 매뉴얼 PDF 또는 문서를 등록한다.
2. 민원 담당자가 "장기 미처리 민원에 대한 안내 문구"를 질문한다.
3. 챗봇은 매뉴얼 근거와 함께 대응 방식, 상담 스크립트, 문서 답변 초안을 제공한다.
4. 담당자는 초안을 검토하고 실제 민원 답변에 반영한다.

#### 수용 기준

- 챗봇 답변은 가능한 경우 관련 매뉴얼 출처를 함께 표시해야 한다.
- 매뉴얼에 근거가 없는 질문은 추정 답변임을 명확히 표시해야 한다.
- 개인정보, 주민등록번호, 연락처 등 민감정보 입력을 제한하거나 마스킹해야 한다.
- 최종 답변 발송 전 담당자 검토가 필요하다는 문구를 제공해야 한다.

### 4.4 뉴스 기사 수집

공공 행정 관련 뉴스를 매일 아침 자동으로 수집하고 요약하는 기능이다.

#### 주요 기능

- 지정 시간 자동 수집
- 키워드 기반 뉴스 검색
- 공공 행정 관련 기사 필터링
- 중복 기사 제거
- 기사 제목, 출처, 발행일, 링크 저장
- 핵심 요약 생성
- 중요도 또는 카테고리 분류
- 아침 브리핑 화면 제공
- 이메일 또는 메신저 알림 연동 옵션

#### 기본 키워드 예시

- 공공행정
- 지방자치
- 민원 서비스
- 행정안전부
- 공공기관
- 전자정부
- 복지 행정
- 재난 안전

#### 수용 기준

- 시스템은 매일 지정된 시간에 뉴스 수집 작업을 실행할 수 있어야 한다.
- 중복 URL 또는 동일 제목 기사는 하나로 합쳐야 한다.
- 사용자는 수집 키워드와 제외 키워드를 관리할 수 있어야 한다.
- 기사 원문 링크를 반드시 제공해야 한다.
- 저작권 보호를 위해 기사 전문을 저장하지 않고 제목, 링크, 요약 중심으로 관리해야 한다.
