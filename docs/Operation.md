# 공공직군 행정업무 슈퍼앱 Operation

## 1. 문서 목적

본 문서는 공공직군 행정업무 슈퍼앱의 실행 방법, 운영 절차, 자주 발생하는 오류, 사용자 사용법을 정리한다. 현재 구현 전 단계이므로 실제 구현 시 적용할 운영 기준과 예상 명령을 함께 기술한다.

## 2. 로컬 실행 기준

### 2.1 Frontend

현재 `frontend/package.json`에는 React, React DOM, Vite, TypeScript 의존성이 포함되어 있다. 개발 서버 실행 스크립트는 아직 정의되어 있지 않으므로 구현 시 다음 스크립트를 추가하는 것을 권장한다.

```json
{
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview",
    "test": "vitest"
  }
}
```

실행 명령:

```bash
cd frontend
npm.cmd install
npm.cmd run dev
```

### 2.2 Backend

`backend` 디렉터리에는 Python 가상환경으로 보이는 `.venv`가 존재한다. 현재 백엔드는 FastAPI/uvicorn이 아니라 Python 표준 라이브러리 `http.server` 기반으로 실행한다.

```bash
cd backend
.\.venv\Scripts\python.exe -m app.main
```

### 2.3 Worker

엑셀 처리, 문서 색인, 뉴스 수집은 시간이 오래 걸릴 수 있으므로 API 프로세스와 분리된 워커에서 실행한다.

```bash
cd backend
.venv\Scripts\activate
celery -A app.workers.celery_app worker --loglevel=info
```

스케줄러 실행 예시:

```bash
celery -A app.workers.celery_app beat --loglevel=info
```

운영 환경에서는 `.env` 파일을 저장소에 커밋하지 않는다.

## 4. 사용자 사용법

### 4.1 팀원 스케줄 관리

1. 로그인 후 캘린더 메뉴로 이동한다.
2. 월간 또는 주간 보기를 선택한다.
3. `일정 추가` 버튼을 눌러 일정 유형, 기간, 제목을 입력한다.
4. 필요 시 공개 범위와 설명을 입력한다.
5. 저장 후 팀 캘린더에서 일정이 표시되는지 확인한다.

운영 팁:

- 휴가와 출장은 일정 유형을 분리해 등록한다.
- 개인정보가 포함된 상세 사유는 입력하지 않는다.
- 장기 부재 일정은 대체 담당자를 설명에 남기는 것을 권장한다.

### 4.2 엑셀 업무 자동화

엑셀 분할:

1. 엑셀 자동화 메뉴에서 `파일 분할`을 선택한다.
2. `.xlsx`, `.xls`, `.csv` 파일을 업로드한다.
3. 미리보기에서 헤더와 컬럼을 확인한다.
4. 분할 기준 컬럼을 선택한다.
5. 실행 후 작업 상태를 확인한다.
6. 완료되면 결과 압축 파일을 다운로드한다.

엑셀 병합:

1. `파일 병합`을 선택한다.
2. 병합할 파일 여러 개를 업로드한다.
3. 헤더 행과 중복 제거 옵션을 선택한다.
4. 컬럼 불일치 경고가 있으면 파일 양식을 확인한다.
5. 실행 후 통합 파일을 다운로드한다.

주의사항:

- 원본 파일은 백업 후 업로드한다.
- 병합 대상 파일의 헤더 구조가 다르면 결과가 달라질 수 있다.
- 주민등록번호, 연락처 등 민감정보가 포함된 파일은 접근 권한을 제한한다.

### 4.3 민원 대응 챗봇

1. 관리자가 최신 민원 매뉴얼을 업로드한다.
2. 색인 상태가 완료인지 확인한다.
3. 민원 담당자는 챗봇 화면에서 질문을 입력한다.
4. 답변 초안과 근거 문서를 함께 확인한다.
5. 담당자가 최종 검토 후 실제 답변에 반영한다.

주의사항:

- 챗봇 답변은 최종 답변이 아니라 검토용 초안이다.
- 매뉴얼 근거가 없는 답변은 별도 확인이 필요하다.
- 민원인의 개인정보를 그대로 입력하지 않는다.

### 4.4 뉴스 기사 수집

1. 관리자가 수집 키워드와 제외 키워드를 설정한다.
2. 시스템은 매일 아침 지정 시간에 기사를 수집한다.
3. 사용자는 뉴스 브리핑 메뉴에서 제목, 출처, 요약, 링크를 확인한다.
4. 필요 시 키워드별로 필터링한다.

주의사항:

- 기사 전문을 복제하거나 재배포하지 않는다.
- 원문 확인이 필요한 경우 제공된 링크로 이동한다.
- 수집 결과가 부정확하면 키워드와 제외 키워드를 조정한다.

## 5. 관리자 운영 절차

### 사용자 및 권한 관리

- 신규 사용자 등록 시 소속 부서와 역할을 지정한다.
- 퇴직 또는 전보 사용자는 즉시 비활성화한다.
- 관리자 권한은 최소 인원에게만 부여한다.

### 민원 매뉴얼 관리

- 매뉴얼 변경 시 새 버전으로 업로드한다.
- 이전 매뉴얼은 삭제하지 않고 보관 상태로 전환한다.
- 색인 실패 시 파일 형식과 텍스트 추출 가능 여부를 확인한다.

### 뉴스 키워드 관리

- 기본 키워드와 제외 키워드를 주기적으로 점검한다.
- 관련 없는 기사가 많이 수집되면 제외 키워드를 추가한다.
- 중요한 정책 이슈가 발생하면 임시 키워드를 추가한다.

### 로그 점검

- 파일 업로드 실패
- 엑셀 작업 실패
- 챗봇 답변 생성 실패
- 뉴스 수집 실패
- 비정상 로그인 시도

## 6. 자주 발생하는 에러와 대응

| 증상 | 원인 | 대응 |
| --- | --- | --- |
| 프론트엔드 실행 실패 | `npm install` 미실행 또는 Node 버전 불일치 | Node LTS 설치 후 `npm install` 재실행 |
| `npm run dev` 없음 | `package.json`에 dev 스크립트 없음 | Vite 스크립트 추가 |
| 백엔드 실행 실패 | 가상환경 미활성화 또는 패키지 누락 | `.venv` 활성화 후 의존성 설치 |
| DB 연결 실패 | `DATABASE_URL` 오류 또는 DB 미실행 | DB 상태와 접속 문자열 확인 |
| 엑셀 업로드 실패 | 파일 확장자/용량 제한 초과 | 허용 형식과 용량 확인 |
| 엑셀 분할 실패 | 기준 컬럼 없음 또는 빈 값 과다 | 컬럼명과 원본 데이터 정리 |
| 병합 결과 이상 | 파일별 헤더 구조 불일치 | 템플릿 통일 후 재업로드 |
| 챗봇 답변 근거 없음 | 매뉴얼 미색인 또는 관련 내용 없음 | 매뉴얼 색인 상태와 문서 내용 확인 |
| 뉴스 미수집 | 스케줄러 중지 또는 API 오류 | 스케줄러 로그와 외부 API 상태 확인 |
| 다운로드 실패 | 결과 파일 만료 또는 저장소 오류 | 작업 재실행 또는 파일 스토리지 확인 |

## 7. 백업 및 복구

### 백업 대상

- PostgreSQL 데이터베이스
- 업로드된 엑셀 원본 파일
- 처리 결과 파일
- 민원 매뉴얼 원본 파일
- 벡터 색인 데이터
- 운영 환경 변수

### 백업 정책

- 데이터베이스는 매일 1회 이상 백업한다.
- 파일 스토리지는 주기적으로 증분 백업한다.
- 민원 매뉴얼은 버전별로 보관한다.
- 백업 복구 테스트를 정기적으로 수행한다.

## 10. 운영 원칙

- 챗봇은 보조 도구이며 최종 행정 판단을 대체하지 않는다.
- 개인정보와 민감정보는 최소한으로 수집하고 필요한 기간만 보관한다.
- 뉴스 기사 원문은 링크로 제공하고 전문을 저장하지 않는다.
- 엑셀 자동화 결과는 사용자가 최종 확인한 뒤 업무에 반영한다.
- 모든 관리자 작업은 감사 로그에 남긴다.
